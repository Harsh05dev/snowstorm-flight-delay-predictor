## Project Overview

Snowstorm Flight Delay Predictor - A machine learning project that predicts flight delays at Newark Liberty International Airport (EWR) during winter weather conditions using weather data (snowfall, temperature, wind) and flight characteristics.

**Tech Stack:** Python, scikit-learn, XGBoost, Streamlit, pandas, matplotlib/seaborn

**Dataset:** 31,121 winter flights (December, January, February) from Newark Airport with weather data from NOAA.

## Architecture Overview

### Data Pipeline Flow
1. **Raw Data** (`data/full_data_flightdelay.csv`) - 6.5M flights from multiple airports
2. **Filtered Data** (`data/ewr_winter_clean.csv`) - 31K Newark winter flights (generated by notebook 01)
3. **Trained Models** (`models/`) - XGBoost, Random Forest, and feature metadata (generated by notebook 02)
4. **Dashboard** (`app/app.py`) - Streamlit app that loads models and provides predictions

### Key Data Processing Steps

**Notebook 01 (01_data_exploration.ipynb):**
- Filters for Newark Liberty International Airport flights only
- Filters for winter months (December=12, January=1, February=2)
- Creates target variable `DELAYED` from `DEP_DEL15` (1 = delayed >15 min)
- Saves cleaned dataset to `data/ewr_winter_clean.csv`

**Notebook 02 (02_modeling.ipynb):**
- Engineers 9 powerful features (HEAVY_SNOW, BELOW_FREEZING, HIGH_WIND, etc.)
- Trains 3 models: Logistic Regression (baseline), Random Forest, XGBoost (best)
- Uses `class_weight='balanced'` / `scale_pos_weight` to handle 74% on-time vs 26% delayed imbalance
- Saves best model (XGBoost, ROC-AUC: 0.732) and artifacts to `models/`

### Feature Engineering

The model uses 22 features total, including 9 engineered features that significantly boost performance:

**Engineered Features:**
- `HEAVY_SNOW`: Flag for snowfall > 1 inch (increases delay rate by 20.6%)
- `BELOW_FREEZING`: TMAX < 32°F
- `HIGH_WIND`: AWND > 15 mph (increases delay rate by 9.8%)
- `HAS_PRECIP`: Any precipitation present
- `SNOW_ON_GROUND`: Snow depth (SNWD) > 0 (increases delay rate by 13.9%)
- `PEAK_HOUR`: Departure in rush hours (7-9 AM, 4-7 PM)
- `SEVERE_WEATHER`: Combo of snow + cold + wind
- `IS_WEEKEND`: Days 6-7 (Saturday/Sunday)
- `CARRIER_DELAY_RATE`: Historical airline delay rate

**Raw Features:**
Weather: SNOW, SNWD, TMAX, AWND, PRCP
Flight: MONTH, DAY_OF_WEEK, DISTANCE_GROUP, PLANE_AGE, CONCURRENT_FLIGHTS, NUMBER_OF_SEATS, AIRPORT_FLIGHTS_MONTH, AIRLINE_AIRPORT_FLIGHTS_MONTH

### Dashboard Architecture (app/app.py)

**Model Loading:**
- Uses `@st.cache_resource` for models (xgb_model, rf_model, feature_columns)
- Uses `@st.cache_data` for datasets (ewr_winter_clean.csv, model_results.csv)
- Paths are relative (`../models/`, `../data/`) - run from `app/` directory

**Prediction Flow:**
1. User adjusts weather sliders (snow, temp, wind) and flight details (airline, time, day)
2. App constructs feature vector with 22 features matching training order
3. For missing context (PLANE_AGE, CONCURRENT_FLIGHTS, etc.), uses median values from dataset
4. XGBoost model returns delay probability (0-1)
5. Displays risk level: >70% = HIGH RISK, 40-70% = MODERATE, <40% = LOW RISK

**Important Detail:** The app uses `feature_columns` list saved during training to ensure correct column order when making predictions.

## Model Performance

Best model: **XGBoost** (200 estimators, max_depth=6, learning_rate=0.1)

Performance metrics on test set (6,225 flights):
- ROC-AUC: 0.732
- Accuracy: 70.2%
- Precision: 44.8%
- Recall: 62.9%
- F1 Score: 52.3%

For comparison:
- Random Forest: ROC-AUC 0.711
- Logistic Regression: ROC-AUC 0.664

## Modifying the Project

**Adding new features:**
1. Add feature engineering in notebook 02 (Step 5)
2. Add feature name to `feature_columns` list (Step 8)
3. Retrain models and save
4. Update `app/app.py` to include new feature in `input_data` DataFrame (lines 105-130)

**Changing the model:**
1. Modify hyperparameters in notebook 02 (cells 12-14)
2. Re-run training cells
3. Save new model with `joblib.dump()`
4. Dashboard automatically uses saved model on next run

**Adding new airports:**
The current implementation is hardcoded for Newark. To add other airports:
1. In notebook 01, filter for additional airport names
2. Ensure weather data exists for that location (PRCP, SNOW, SNWD, TMAX, AWND columns)
3. Re-run full pipeline

## Data Insights

Key findings from EDA (notebook 01):
- Snow increases delay rate from 24.2% (no snow) to 37.6% (snowing) - 1.6x multiplier
- Heavy snow (>1") causes 46% delay rate vs 25.4% baseline
- High winds (>15 mph) increase delays by 9.8 percentage points
- Dataset is imbalanced: 74% on-time, 26% delayed (handled via class weighting)

Top delay predictors (by XGBoost feature importance):
1. CARRIER_DELAY_RATE (airline historical performance)
2. AIRLINE_AIRPORT_FLIGHTS_MONTH (airline activity at airport)
3. Weather-related features (HEAVY_SNOW, SNOW_ON_GROUND, HIGH_WIND)
